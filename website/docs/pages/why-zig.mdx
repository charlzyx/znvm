# Why Zig?

Choosing Zig to build a **node version manager** wasn't about using the newest tool. It was a strategic decision to solve **nvm-slow** and achieve **extreme performance** with **zero overhead**.

## The Problem with Shell-based Managers

Most traditional Node.js version managers like **nvm** are written entirely in Shell. While Shell is flexible, it's slow. Every time you open a new terminal tab, your shell has to source hundreds of lines of code, which can add hundreds of milliseconds to your startup time. This is the core of the **nvm-slow** problem.

## The Problem with Feature-rich Managers

Newer managers like **fnm** (written in Rust) are much faster, but they often try to do too much. They handle multiple platforms (including Windows), manage various configuration files, and sometimes require their own runtime. This adds complexity and "bloat" to a tool that should be invisible.

## The Zig Solution

Zig allows us to build a high-performance **node-version-manage** tool that:

- **Is Blazingly Fast**: `znvm` starts in less than 5ms. It's so fast you won't even know it's there.
- **Has Zero Dependencies**: `znvm` compiles to a tiny, standalone binary.
- **Is Unix-Optimized**: By focusing on Unix-like systems (macOS and Linux), we can use native system calls and avoid the complexity of cross-platform abstractions.
- **Is Predictable**: Zig's focus on memory safety and explicit control ensures that `znvm` behaves exactly as you expect.

## Core Philosophy: Zero-Overhead

Our goal is to make Node.js version management a "background" task. You should never have to wait for your version manager. With Zig, we've achieved a level of performance that makes version switching feel instantaneous.
