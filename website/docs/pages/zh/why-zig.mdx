# 为什么选择 Zig?

选择用 Zig 来构建 **Node.js 版本管理器** 并非为了追求新鲜感，而是为解决 **nvm-slow** 问题、实现 **极致性能** 和 **零负担** 的战略性决策。

## Shell 管理器的痛点

大多数传统的 Node.js 版本管理器完全由 Shell 编写。虽然 Shell 灵活，但运行缓慢。每次您打开一个新的终端标签页，您的 shell 都必须加载数百行代码，这可能会为启动时间增加数百毫秒。这就是 **nvm-slow** 问题的核心。

## 功能冗余的管理器

使用 Rust 编写的新一代管理器如 **fnm** 速度快得多，但它们往往试图承载过多功能。它们需要处理多平台（包括 Windows）、管理各种配置文件，有时还需要独立的运行时。这为一个本应"隐形"的工具增加了复杂性和"臃肿感"。

## Zig 的解决方案

Zig 让我们能够构建一个高性能的 **node-version-manage** 工具：

- **极致速度**: `znvm` 启动时间小于 5ms。快到您甚至感觉不到它的存在。
- **零依赖**: `znvm` 编译为微型、独立的二进制文件。
- **Unix 优化**: 通过专注于 Unix 类系统（macOS 和 Linux），我们可以使用原生系统调用，避开跨平台抽象带来的复杂性。
- **可预测性**: Zig 对内存安全和显式控制的关注，确保了 `znvm` 的行为完全符合预期。

## 核心哲学：零负担 (Zero-Overhead)

我们的目标是让 Node.js 版本管理成为一项"后台"任务。您永远不应该为了版本管理器而等待。通过 Zig，我们实现了让版本切换感觉像是瞬间完成的性能水平。
